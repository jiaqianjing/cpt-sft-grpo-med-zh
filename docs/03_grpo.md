# 阶段 03 · GRPO（强化学习，可验证奖励）

> 一句话：让模型"自己多试几次，答对的给奖励"——用强化学习把答对率往上推。

## 1. 喂的是什么数据

**带可验证答案的选择题**（只要能自动判对错就行）。共 30k 条（本次用 12k），来自 MedQA-zh + CMExam
的训练集。注意：**只给题目，不给参考推理**——推理由模型自己生成，对错由程序判定。真实样本：

```
问题：卧位腰椎穿刺，脑脊液压力正常值是（　　）。
A. 80～180mmH2O   B. 50～70mmH2O   C. 230～250mmH2O   D. 260～280mmH2O
（可验证的标准答案：A）
```

## 2. 训练目标（机制）：可验证奖励

对每道题，模型采样 **G=8 个**不同的回答；每个回答按规则打分（纯函数，`med_pipeline/tools/reward.py`）：

```
reward = 1.0 × 答对(exact-match 抽取 \boxed{} == 标准答案)   +   0.2 × 格式正确(恰好一个 \boxed{})
```

GRPO 用**组内相对优势**（同一题 8 个回答里，比平均好的被强化、差的被抑制）做策略梯度更新。
**没有参考答案可模仿**——模型完全靠"答对得分"这个信号自我改进。

## 3. 进步体现在哪：训练奖励明显上升

训练过程中，"答对率"奖励稳定上升（W&B 有完整曲线）：

| | 训练开始 | 训练结束 |
|---|:---:|:---:|
| correctness 奖励（答对率） | **0.40** | **0.625** |
| 总奖励 | 0.46 | 0.82 |

**这说明**：GRPO 的机制**确实在生效**——在它看到的题目上，模型学会了更多地答对。KL 全程很低
（≈6e-4），说明没有跑偏成乱码。

## 4. 诚实说明（训练 vs 测试的差距）

- **训练奖励涨了（0.40→0.625），但迁移到留出测试有限**：R4 生成准确率 0.552，仅比 R3（0.537）高 +0.015，
  且**仍低于 base（0.589）**。
- 真实的两面例子（同一批测试题）：
  - ✅ 某题 R3 答对 B、R4 保持答对 B（GRPO 没破坏对的）。
  - ❌ 某题 R3 本来答对 C，R4 **过度推理**（输出从 1068 字涨到 1190 字）反而绕到错误的 D。
- 根因：GRPO 是从一个**已被弱 SFT 拖偏、爱啰嗦**的起点出发的；RL 在训练题上提了分，但"过度推理"
  的毛病没根治，泛化天花板被 SFT 基座限制住了。**先用强教师把 SFT 做正，GRPO 的测试增益才容易显现。**
