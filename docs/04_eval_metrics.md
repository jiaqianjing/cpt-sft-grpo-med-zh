# 评测指标详解：Perplexity vs Knowledge vs Generative

本项目用**三条评测轴**衡量每个阶段的增益，它们测的是不同维度的能力：

| 指标 | 一句话 | 需要模型生成文本吗 | 有对错之分吗 |
|------|--------|:---:|:---:|
| **Perplexity**（困惑度） | 模型对领域文本有多"熟悉" | ❌ | ❌ 只有高低 |
| **Knowledge**（知识准确率） | 模型能不能判断出正确选项 | ❌ | ✅ |
| **Generative**（生成准确率） | 模型能不能写出推理并给对答案 | ✅ | ✅ |

---

## 1. Perplexity（困惑度）

### 含义

给模型一段**连续的医学文本**（不是问答），计算模型对每个 token 的预测概率，取指数平均。
衡量的是：模型对这类文本有多"熟悉"、读得多"顺"。**没有对错之分，只有熟不熟。**

### 公式

$$PPL = \exp\left(-\frac{1}{N}\sum_{i=1}^{N}\log P(t_i \mid t_1,...,t_{i-1})\right)$$

### 数值举例

输入文本：`"补阳还五汤加减黄芪40g"` → tokenize 为 $[t_1, t_2, ..., t_{10}]$

| 位置 | token | P(token\|前文) | log P |
|------|-------|:---:|:---:|
| 1 | 补 | 0.02 | −3.91 |
| 2 | 阳 | 0.15 | −1.90 |
| 3 | 还 | 0.25 | −1.39 |
| 4 | 五 | 0.60 | −0.51 |
| 5 | 汤 | 0.80 | −0.22 |
| 6 | 加 | 0.30 | −1.20 |
| 7 | 减 | 0.45 | −0.80 |
| 8 | 黄 | 0.10 | −2.30 |
| 9 | 芪 | 0.70 | −0.36 |
| 10 | 40g | 0.20 | −1.61 |

$$PPL = \exp\left(-\frac{-3.91-1.90-1.39-0.51-0.22-1.20-0.80-2.30-0.36-1.61}{10}\right) = \exp(1.42) = 4.14$$

### 本项目结果

在整个留出医学文本集上：base PPL=**7.04**，CPT 后 PPL=**5.79**（↓1.25，约 −18%）。
数字越低 = 每一步犹豫的候选越少 = 模型对领域文本越熟悉。

### 类比

你拿起一本中医教材，读得越顺畅（不用停下来查词），说明你对这个领域越熟悉。

## 2. Knowledge（loglikelihood MCQ）

### 含义

给一道选择题，模型**不生成任何文字**——而是对每个选项分别计算"接在题目后面有多顺"的概率，
选概率最高的作为答案。有明确的对/错。

### 公式

对每个选项文本 $[t_1, ..., t_L]$，计算长度归一化的对数概率：

$$\text{score}(选项) = \frac{1}{L}\sum_{j=1}^{L}\log P(t_j \mid \text{题目}, t_1,...,t_{j-1})$$

选 score 最高的选项 → 与标准答案比对 → 判断对错。

### 数值举例

题目 prompt：`"卧位腰椎穿刺，脑脊液压力正常值是"`

| 选项 | 文本 | tokens 数 | 各 token 的 log P | score (平均) |
|------|------|:---:|---|:---:|
| A | 80～180mmH₂O | 5 | −0.5, −0.3, −0.4, −0.2, −0.3 | **−0.34** |
| B | 50～70mmH₂O | 4 | −1.2, −0.8, −0.9, −0.7 | −0.90 |
| C | 230～250mmH₂O | 5 | −1.5, −1.1, −0.9, −0.8, −0.6 | −0.98 |
| D | 260～280mmH₂O | 5 | −1.8, −1.3, −1.0, −0.9, −0.7 | −1.14 |

选 score 最高的 → **A**（−0.34 > −0.90 > −0.98 > −1.14）。标准答案也是 A → ✅ correct=1。

对所有测试题求平均正确率 → knowledge accuracy。

### 本项目结果

base knowledge = **0.661**；CPT-only = 0.636（略降）；全链路 R4 = 0.646。

### 类比

完形填空——不用你写出答案，只问你"哪个选项填进去最通顺"。

## 3. Generative（生成准确率）

### 含义

模型**真的写出一段推理过程**，最后给出 `\boxed{A}`。用正则从生成文本中抽取答案字母，判对错。
这是模型作为"答题者"的完整端到端表现。

### 流程

```
输入：题目 + 选项 + "请逐步推理，并在最后给出 \boxed{} 形式的答案。"
      ↓
模型 greedy 生成 ≤768 tokens
      ↓
正则抽取 \boxed{X} 中的字母
      ↓
与标准答案 exact-match → 1 or 0
      ↓
所有题目求平均 → generative accuracy
```

### 本项目结果

| Run | generative |
|-----|:---:|
| R0 base | **0.589** |
| R1 SFT | 0.544 |
| R3 CPT+SFT | 0.537 |
| R4 +GRPO | 0.552 |

R2（CPT-only）未跑 generative——CPT 不改变生成行为，跟 R0 无实质区别。

### 类比

考试时你要写出完整的解题过程和最终答案。

## 三者关系总结

```
              ┌─────────────────────────────────────────────┐
              │       模型参数里编码的"领域知识"              │
              └────────────┬─────────────────┬──────────────┘
                           │                 │
              ┌────────────▼────┐   ┌────────▼──────────────┐
              │  Perplexity     │   │  Knowledge            │
              │  "读得顺不顺"   │   │  "能判断对错吗"        │
              │  (无标准答案)    │   │  (有标准答案,不生成)   │
              └─────────────────┘   └───────────────────────┘
                                             │
                                    + 生成能力 + 格式遵循
                                             │
                                    ┌────────▼──────────────┐
                                    │  Generative           │
                                    │  "能写出推理并答对吗"   │
                                    │  (有标准答案,要生成)    │
                                    └───────────────────────┘
```

### 为什么每个阶段主要看不同的指标

| 阶段 | 主信号 | 原因 |
|------|--------|------|
| CPT | **Perplexity** | CPT 让模型"读顺领域文本"，困惑度保证下降 |
| SFT | **Generative** | SFT 教的是"怎么组织输出"，要看真实生成效果 |
| GRPO | **Generative** | GRPO 优化的就是"生成后答对"这个目标 |
| 所有阶段 | Knowledge | 作为通用参考，base 也能测，跨阶段可比 |

---

## 为什么 R2 (CPT-only) 跑了 Knowledge 但跳过了 Generative

- **Knowledge 对 CPT 有意义**：CPT 改变了模型内部的知识分布，可能影响"哪个选项续写概率更高"。
  实测从 0.661 降到 0.636——说明 TCM 语料偏移了分布，但没提升选择判断力。
- **Generative 对 CPT 无意义**：CPT 不改变"怎么组织输出"的行为，R2 和 R0 在生成风格上几乎一样，
  跑出来不会比 R0 多告诉你什么，所以设计上跳过。
